# Guia de Contribuição 🤝

Primeiro, obrigado por se interessar em contribuir para o **Sortudo**! É o esforço comunitário que torna projetos open-source incríveis.

## 🌈 Como posso ajudar?

- **Relatando Bugs**: Encontrou algo errado? Abra uma issue com detalhes para reprodução.
- **Sugerindo Funcionalidades**: Tem uma ideia para uma nova visualização estatística? Adoraríamos ouvir.
- **Melhorando a Documentação**: Correções de texto ou melhores explicações são sempre bem-vindas.
- **Escrevendo Código**: Veja as issues abertas ou sugira uma melhoria técnica (refatoração, testes, performance).

## 🛠️ Configuração de Desenvolvimento Local

1.  **Fork e Clone**: Faça o fork e clone para sua máquina.
2.  **Branches**: Crie uma branch descritiva a partir da `main`.
    - `feat/nome-da-feature`
    - `fix/nome-do-bug`
    - `docs/melhoria-na-doc`
3.  **Dependências**: Use `bun install` (ou `npm install`).
4.  **Lint e Formatação**: Certifique-se de que seu código segue as regras do ESLint configuradas no projeto.

## 📝 Padrões de Commit

Tentamos seguir o [Conventional Commits](https://www.conventionalcommits.org/):

- `feat`: Uma nova funcionalidade.
- `fix`: Correção de um bug.
- `docs`: Mudanças apenas na documentação.
- `style`: Mudanças que não afetam o significado do código (espaço em branco, formatação).
- `refactor`: Mudança de código que não corrige um bug nem adiciona uma funcionalidade.
- `test`: Adicionando testes ausentes ou corrigindo testes existentes.

## 🚀 Processo de Pull Request

1.  Certifique-se de que o projeto ainda constrói corretamente (`bun run build`).
2.  Descreva claramente suas mudanças no corpo do PR.
3.  Aguarde a revisão. Estamos ansiosos para ver seu trabalho!

---

Ao contribuir, você concorda que suas contribuições serão licenciadas sob a licença MIT do projeto.
